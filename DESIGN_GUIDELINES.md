# 設計規範與開發指南

## 概述

本文檔規定系統的設計、風格、元件使用方式，確保所有頁面保持一致的使用者體驗。

---

## 核心原則

### 1. 設計系統為單一來源 (Single Source of Truth)

- 所有設計 tokens（字級、寬度、間距）定義在 `lib/design-system.ts`
- 新開發頁面必須導入並使用 design-system 中的常數
- 避免在各頁面中硬編碼樣式值

### 2. 區分「核心」與「建議」規範

**核心規範（CORE）：** 不應違反，若需要例外必須：
- 說明充分理由
- 向產品經理確認
- 更新本文檔記錄例外情況

**建議規範（RECOMMENDED）：** 預設方案，可根據特定需求彈性調整

### 3. 元件優先於樣式

新需求時：
1. 先評估現有元件是否可透過 props/variant 滿足
2. 若需要新樣式，優先擴展現有元件而非建立新元件
3. 預留擴展接口（className、variant、size 等）

## 字級規範

### 核心規範

- **最小字級：16px（text-base）**
- 不可使用 `text-xs`（12px）或 `text-sm`（14px）在主要內容
- 年長使用者必須能清晰閱讀所有內容

### 推薦字級配置

| 用途 | 字級 | Tailwind 類別 |
|------|------|-------------|
| 輔助資訊（時間戳記） | 14px | `text-sm` |
| 正文、表格、按鈕、標籤 | 16px | `text-base` |
| 卡片標題、表單標籤 | 18px | `text-lg` |
| 頁面副標題 | 20px | `text-xl` |
| 頁面主標題 | 24px | `text-2xl` |

**何時使用例外（14px）：**
- 只能用於時間戳記、版本號、極小輔助資訊
- 必須添加註解說明為何使用

## 容器寬度規範

### 核心規範

- **統一頁面寬度：max-w-6xl（1152px）**
- 新頁面使用 `components/layout/page-container.tsx` 的 `PageContainer` 元件
- 既有頁面多數仍自行拼版型字串，**逐步遷移中**（見 `docs/todo-design-corrections.md`）

### 使用方式

```tsx
import { PageContainer, PageHeader } from "@/components/layout/page-container"

export default function Page() {
  return (
    <PageContainer>
      <PageHeader title="頁面標題" description="說明文字">
        {/* 右側操作按鈕 */}
      </PageHeader>

      {/* 頁面內容 */}
    </PageContainer>
  )
}
```

`PageContainer` 接受 `width="narrow" | "standard" | "wide"`，對應下表三種寬度，預設 `standard`。

### 建議寬度配置

| 情境 | 寬度 | 說明 |
|------|------|------|
| 表單、單欄 | max-w-4xl（896px） | 較窄的內容適用 |
| 列表、儀表板 | max-w-6xl（1152px） | **推薦、預設** |
| 數據密集頁面 | max-w-7xl（1280px） | 複雜表格適用 |

## 導航規範

### 核心規範

1. **無第二層導航**
   - 一級導航（填報專區、審查專區、管理專區）之下不設二級導航
   - 各階段頁面獨立呈現，不需內層 tabs 進行切換

2. **頁面返回邏輯**
   - **頂層功能頁**（如 /filing、/admin/user-management）：無「返回首頁」按鈕
   - **子詳情頁**（如 /filing/[id]）：顯示「返回上一頁」按鈕

## 間距規範

### 推薦配置

| 用途 | Tailwind 類別 | 像素值 |
|------|-------------|--------|
| 卡片間距 | `gap-4` | 16px |
| 卡片內邊距（標準） | `p-4` | 16px |
| 卡片內邊距（舒適） | `p-5` | 20px |
| 版塊間距 | `space-y-6` | 24px |
| 元素間距 | `gap-2` | 8px |

### 公告卡片優化

- 使用 `p-4` 而非 `p-6`，減少內部留白
- 卡片間距 `gap-3`（12px）確保視覺緊湊
- 不固定卡片高度，使用 `h-auto`
- 最小高度 `min-h-24`（96px）保留可點擊面積

## 元件設計指南

### Tabs 元件

- **預設樣式：下劃線（underline）**
- 系統內所有 Tabs 應保持一致風格
- 若需要 pill 或其他樣式，需說明理由並與團隊確認

### Badge 與標籤

- 統一大小：`text-base px-2.5 py-1`
- 顏色使用 design-system 中定義的配色
- 避免自訂大小或顏色，除非有特殊需求

### 表格

- 行高：`h-14`（56px）增加可點擊面積
- 字級：`text-base`（16px）確保可讀性
- 內邊距：`px-4 py-3` 統一單位格間距

## 新頁面開發檢查清單

開發新頁面時：

- [ ] 使用 `PageContainer` 包裝頁面內容
- [ ] 使用 `PageHeader` 定義頁面標題
- [ ] 字級最小 16px（text-base）
- [ ] 所有間距使用 design-system 中的推薦值
- [ ] Tabs 元件使用下劃線樣式
- [ ] 導航遵守「無二級導航」規範
- [ ] 頂層頁面無「返回首頁」按鈕
- [ ] 使用 `max-w-6xl` 限制容器寬度

## 規範修改流程

若需要調整設計規範：

1. **評估原因** - 是否確實無法用現有規範滿足？
2. **提出方案** - 向產品經理說明需要調整的內容和理由
3. **確認** - 確認修改無誤後才更新 design-system.ts
4. **記錄** - 更新本文檔說明修改原因和生效日期

### 不主動修改的情況

- 特定頁面需要例外樣式 → 用 className 覆寫，不修改系統規範
- 新元件需要不同樣式 → 先看現有元件是否可擴展

### 主動告知修改的情況

- 新需求明確違反核心規範（如要求 12px 字級）
- 多個頁面出現相同「例外」需求 → 代表規範需調整
- 現有規範造成開發困難或使用者體驗問題

## 常見問題

### Q: 為什麼最小字級是 16px 而不是 14px？

A: 考慮使用者年紀較大，16px 能提供更好的可讀性和易用性。14px 只允許在極少數輔助資訊中使用。

### Q: 新頁面需要特殊寬度怎麼辦？

A: 先評估 max-w-6xl 是否可用。若確實需要其他寬度，說明理由後可使用 `max-w-4xl` 或 `max-w-7xl`。

### Q: 現有頁面寬度不一致怎麼辦？

A: 所有頁面應逐步遷移至 `PageContainer` 元件，使用統一寬度。
目前只有公告管理模組（`/announcement-management` 各頁）已遷移。

### Q: 下拉選單為什麼會長短不一？

A: 本專案 shadcn 版本的 `SelectTrigger` 預設是 `w-fit`（不是 `w-full`）。
放進 grid 或需要對齊時，**務必自行指定寬度**（`w-full` 或固定寬），
否則會出現各欄長短不一的參差版型。

### Q: 能否修改核心規範？

A: 核心規範應盡量穩定。若必須修改，需與產品經理充分討論並更新本文檔。
