## 角色

| 角色 | 說明 |
|------|------|
| 醫事司 | 主管機關。外加容額全程由其操作；容額填報的公告者；容額成果報告的確認/歸檔者 |
| 醫策會 | 醫策會。容額填報的初審與分組會議、RRC 安排；外加容額成果報告的協同審查方之一 |
| RRC | 專科醫師訓練計畫認定會。容額填報的大會審查 |
| 醫學會 | 各專科醫學會（25 個）。容額填報與容額成果報告的填報方 |
| 訓練醫院 | 外加容額的申請方（系統外發函）；外加容額成果報告的提交方（系統外發函） |

## 容額填報 ✅

醫事司發起、醫學會填報、多階段審查、最後公告的主流程。

### 狀態機

```
醫事司於管理專區設定「容額填報開放」
      ↓（開放後醫學會才可填報）
醫學會填報 → 送件
      ↓
醫策會初審
      ↓（通過）
分組會議審查（醫策會內部安排）
      ↓
RRC 大會審查
      ↓
醫策會更新大會審查結果 → 推進至「待公告」
      ↓                         ↘（同時）醫學會需辦理「容額成果報告」（見下節）
醫事司公告 → 已公告
```

### 退件規則 ✅（重要）

- 任一階段皆可退件，案件**直接退回醫學會**修正。
- 醫學會重新送件後，**由退件的單位接續審查**，不重走整條鏈。
  - 例：案件已過 RRC、進到醫事司，醫事司退件 → 醫學會修正重送 → **直接回到醫事司**，不再走「醫策會 → 分組會議 → RRC」。
- 實作含義：案件需記錄「退件返回的階段（resumeStage）」，重送後跳回該階段，而非從頭。

### 成果報告與公告的關係 ✅

- 案件進「待公告」時，醫學會**同時**需辦理容額成果報告。
- 但成果報告**不影響醫事司的公告作業**，屬內部文件歸檔流程，兩者平行。

## 容額成果報告 ✅

> ⚠️ **命名陷阱（見「我預計要做的事」#1）**：客戶要求維持「容額成果報告」此 wording，
> 但其核心意義**不是**容額執行成效，而是「**容額申請於 RRC 審查後的審查細節補充資訊**」。
> 設計時勿被標題誤導。

### 流程 ✅

```
容額填報案件進「待公告」
      ↓
醫學會上傳審查細部資料 → 送出（與執行成效無關，上傳送出即可）
      ↓（直接給醫事司，不經醫策會）
醫事司：確認 → 歸檔　／　有需要 → 退回補件
```

- **無醫策會參與**。醫學會 ↔ 醫事司 直接往來。
- 沒有「執行成效」的評估，只是補件、確認、歸檔。
- 依附在容額填報案件上，是待公告後的子任務，非獨立填報單元。

### 上傳與審查的surface分離 ✅（權限邊界＝路由）

- **上傳（填報專區，醫學會）**：掛在容額填報案件內容頁的「待公告」子區塊。
- **確認/退回（審查專區，醫事司）**：獨立surface「容額成果報告審查」，醫事司看送來的
  補件並確認歸檔或退回補件。醫學會不能確認自己上傳的報告，故上傳與審查分屬兩路由。
- 由現有 `/review/outcome-report`（「成果報告審查」）改造而成，**並移除其中的「醫策會初審」
  中間步驟**（容額成果報告不經醫策會，原實作建錯）。
- 審查專區子導覽：填報審查｜醫院容額分配審查｜**容額成果報告審查**。

### ⚠️ 與現有實作的落差

現有 `/filing/outcome-report` 的「容額成果報告」tab 建成「醫策會上傳＋初審 → 醫事司審查」，
**與本節相矛盾**，需重做為「醫學會上傳 → 醫事司確認/退回」，並改依附於容額填報案件。

## 外加容額管理 ✅

登錄 → 審查 → 公告，全程醫事司單一角色。已於 `feat/additional-quota-upload-part2` 實作。

### 狀態機

```
醫事司登錄申請（醫院來文）→ 待審查
      ↓（內部會議後登錄審查結果）
待公告
      ↓（公告作業跨多院彙整，於公告管理辦理）
已公告
```

- 唯一鍵為申請項目 uid（同院可在不同分科各自申請）。
- 容額上限為硬限制：核定容額 + 申請容額 > 上限即不得送出。
- 分類原則為可維護的選項，升級為 `{ 名稱, 需成果報告: on/off }`；「需成果報告」開關
  決定該案公告滿一年後是否需提交外加容額成果報告（見下方「外加容額成果報告」）。
- 公告為跨案件的滾動彙整（全年隨時可再申請、再公告加入同一份公告），**非單案欄位**，
  由 `announcement-management` 辦理。完整滾動公告設計待全系統完成後再評估。💡

## 外加容額成果報告 ✅

> 外加容額案件**公告執行滿一年後**，需訓練醫院提交成果報告。

### 適用範圍 ✅

- **只有特定「分類原則」的外加容額案件**需要提交（非全部）。目前為「支援偏鄉」。
- **由分類原則的「需成果報告」開關決定，不以名稱硬比對** —— 分類原則選項升級為
  `{ 名稱, 需成果報告: on/off }`，維護彈窗每項附一個開關。避免改名導致追蹤列表失聯。
- 觸發時點：案件公告後滿一年。

### 權限邊界與模組歸屬 ✅

- 外加容額的**申請＋審查**全程醫事司單一角色（合併於外加容額管理頁）。
- 但成果報告涉及**醫策會**。若把醫策會評論欄塞進外加容額管理頁，後端授權會外溢
  （醫策會將能做容額填報＋審查），非所欲。
- **決策：外加容額成果報告為獨立模組（自己的路由與 nav 入口）**，醫事司與醫策會皆授權
  進此模組，授權不外溢到外加容額管理。權限邊界＝路由，非頁面內欄位級。
- 外加容額管理的案件頁僅放唯讀「本案需成果報告 → 前往」連結；可編輯評論在成果報告模組。
- 現有 nav 的「成果報告上傳」入口**保留但收斂為「外加容額成果報告」**
  （容額成果報告改依附容額填報案件，離開此入口）。

### 流程 ✅

```
訓練醫院系統外發函 → 醫事司 ＋ 醫策會（兩單位皆收到）
      ↓（兩單位於系統外平行協調分工）
醫事司、醫策會分工審查 → 各自留下評論 → 儲存歸檔
```

- **沒有「不通過 / 退回」的審查狀態**，只有：審查完成 → 留評論 → 歸檔。
- 用途：該醫院**後續年度再申請外加容額時，作為審查評斷的依據**。
- 涉及醫策會（外加容額的申請/審查本身不涉及醫策會，僅此成果報告涉及）。

### ⚠️ 與現有實作的落差

現有 `/filing/outcome-report` 的「外加容額成果報告」tab 與 `/review/outcome-report`
建成「醫策會上傳＋初審 → 醫事司審查」，需重做為「醫事司與醫策會平行、只留評論、無退回」，
並掛在具備適用分類原則、且公告滿一年的外加容額案件上。

## 容額微調 💡（規劃中，細節待確認）

- 大方向：一切審核公告完成後，**全年度**醫學會仍可能額外送件申請容額數字的**微調**。
- 對架構的要求：容額填報的案件模型**不應把「已公告」寫死為終態不可變**；
  需保留讓已公告案件衍生「微調」子請求、走（可能較輕量的）審查路徑的擴充空間。
- 建議：階段以資料驅動（stage config），避免日後加微調需重構。
